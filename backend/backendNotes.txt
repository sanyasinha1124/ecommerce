If a user's browser gets stuck in a loop calling our 'Event Details' API, how does the Rate Limiter help?"
"The Rate Limiter acts as a circuit breaker. It ensures that one client's infinite loop doesn't consume all the server's threads or database connections. By returning a 429 error, it preserves resources for other users. On the frontend, I would see those 429s in the Network tab, which immediately tells me my Angular code is firing too many requests."

const apiLimiter = rateLimit({
	windowMs: 15 * 60 * 1000, // 15 minutes
	max: 100, // Limit each IP to 100 requests per window
	message: 'Too many requests, please try again later.',
    // This executes BEFORE your router logic
});

// Apply to your booking route

If a request comes in and multiple things are wrong:

Rate Limit Exceeded (429): Checked First(first line of defence). If the limit is hit, nothing else matters.

Authentication/Authorization (401/403): Checked Second.

Input Validation (400): Checked Third.

Business Logic Errors (Infinite Loops/500): Checked Last.


-- In SQL Transactions
In a database transaction  errors are usually handled by the database engine's Parser and Executor.

Syntax Errors: Treated first (the query won't even run).

Constraint Errors: (e.g., Foreign Key or Unique violations). If multiple rows violate constraints, the DB engine usually stops at the first one it tries to process and triggers a ROLLBACK.

-- 1. In Sequential Code (Try-Catch)
In a standard try...catch block, the very first error that is encountered stops the execution of the entire block immediately.

Priority: The first line of code that fails.

Behavior: Any errors that would have happened on the lines below it are never even reached.

TypeScript
try {
  const a = undefined;
  console.log(a.name); // 1. ERROR HAPPENS HERE (Execution stops)
  
  const b = 10 / 0;    // This line is never reached
} catch (error) {
  console.log("Only the first error is caught:", error);
}

guards in backend - A guard is a function that has access to the Request, Response, and a next() function. If the guard is satisfied, it calls next(); if not, it sends a 401 Unauthorized or 403 Forbidden response immediately.

const router = Router();
